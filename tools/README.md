# tools/

Canonical, reusable reverse-engineering infrastructure. (One-shot
investigation scripts live in [`experiments/`](experiments/).)

| Script | Purpose |
| --- | --- |
| `instrument_encrypt_entry.py` | Node+jsdom harness that patches the live WASM to dump the n-token encrypt entry's args (key ptr / plaintext / length) at prologue + epilogue. The base harness most captures build on. |
| `find_ntoken_aes.py` | Locates the n-token AES encrypt entry structurally (no hard-coded indices). |
| `refine_hsw_labels.py` | Regenerates the per-function role map (`docs/hsw_function_labels.json`). |
| `backtest.py` | Backtests extraction across historical builds. |

The validated fixslice AES (RustCrypto `fixslice32` port, == the bundle's AES)
now lives in the package as `hcaptcha.tools.fixslice`. The n-token master-key
recovery is `hcaptcha.hsw_n_token_decrypt.recover_ntoken_master_live()`;
end-to-end solve + third-party proof are in [`experiments/`](experiments/).

All capture/result artifacts (`*.json`, `*.run.log`) are git-ignored —
they are regenerated by re-running the scripts against the current build.
